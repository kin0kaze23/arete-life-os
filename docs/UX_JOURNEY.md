# UX Journey (End-to-End)

## Daily loop (core journey)

### Morning (Plan)

- Dashboard shows:
  - Do Today: Top 3 priorities (what/why/how)
  - Watch Today: Top 3 watch-outs (why/impact/prevention)
- User taps "Plan My Day" to generate time-block plan with buffers/rest.

### Daytime (Capture + Prepare)

- User adds future event via Log Bar ("Korea trip Mar 2026").
- System generates:
  - milestones (T-60/T-30/T-14/T-7), tasks, watch-outs
- User uploads memories via Log Bar (docs/photos/receipts/health).
- System auto-structures into Memory Vault and updates recs/risk.

### Night (Reflect)

- 2-minute nightly review:
  - done/not done + reason
  - energy/mood pulse
  - quick family/spiritual prompt
- Feedback updates tomorrow’s plan and recommendation thresholds.

## Key UX principles

- Minimal clicks: Dashboard answers "Do + Watch" immediately.
- Calm UX: show only Top 3 + View all (avoid overwhelming feeds).
- Reactive system: all updates propagate everywhere without refresh.
- Trust: everything links to sources; uncertainty -> Needs Review queue.

## Primary entry points

- Dashboard: decisions + action
- Log Bar: capture + command
- Needs Review: resolve uncertainty quickly
